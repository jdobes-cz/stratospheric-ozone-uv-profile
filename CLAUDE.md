## Project Overview

Stratospheric balloon payload project for measuring ozone and UV radiation profiles at different altitudes. The project has two main parts:

1. **Python pipeline (WSL)**: Downloads CAMS ozone forecast data, interpolates it, and runs libRadtran radiative transfer simulations to compute UV irradiance profiles at 0–40 km altitude.
2. **Arduino firmware**: Data logger for an Arduino Pro Mini (ATmega328PB, 3.3V/8MHz) that reads UV (LTR390), pressure/temperature (MS5611), ozone (SEN0321), and RTC (DS3231) sensors, storing records to AT24C256 EEPROM and microSD.

## Environment & Commands

### Python (must run in WSL)

- **Interpreter**: `./py <script>` (wrapper for the `.venv` Python). Never use bare `python` or `python3`.
- **Packages**: `./pip install -r requirements.txt`. Never use bare `pip`.
- **Working directory**: Always run from repo root.

### Data pipeline steps (WSL)

```bash
# 1. Download CAMS ozone forecast (requires ~/.cdsapirc with API key)
./py cams_download3.py

# 2. Interpolate ozone profile to target coordinates → o3_mmr.dat
./py interpolation1.py

# 3. Run libRadtran UV simulations at 0–40 km altitude
./run_loop.sh
```

### Arduino

The sketch is `Arduino/Arduino.ino`. Compile and upload via Arduino IDE or `arduino-cli` targeting ATmega328PB at 3.3V/8MHz. Required libraries: LTR390, MS5611, DFRobot_OzoneSensor, RTClib, SD.

## Architecture

### Python pipeline flow

`cams_download3.py` → downloads NetCDF from CAMS API → `interpolation1.py` reads it, interpolates to a lat/lon point, converts pressure levels to altitude (ISA 1976 piecewise model up to 47 km), and writes `o3_mmr.dat` (libRadtran mol_file format with pressure coordinate) → `run_loop.sh` iterates 0–40 km, substituting `zout_sea` in `uvspec_template.inp` and running libRadtran's `uvspec` solver → outputs `loop/eup_*km.dat` files.

### Arduino firmware

Single-file sketch (`Arduino/Arduino.ino`) with no external state machine. All sensors share I2C; microSD uses SPI on CS=D10. Dual storage: EEPROM (primary, 34-byte binary records with 8-byte header) and microSD (backup, daily CSV files named `YYYYMMDD.CSV`). Serial command interface at 9600 baud for `record`, `stop`, `dump`, `dumpsd`, `status`, `clear`, `settime`. Avoids the Arduino `String` class entirely—uses char buffers and `F()` macro for flash strings to minimize RAM usage.

### Key data files

- `uvspec_template.inp`: libRadtran input template; `run_loop.sh` substitutes `zout_sea` per altitude step.
- `o3_mmr.dat`: Ozone mass mixing ratio profile (pressure vs. O3 kg/kg), generated by `interpolation1.py`, consumed by libRadtran.
- `ltr-390uv-01-response/`: LTR390 sensor spectral response data and overlay scripts for characterizing the UV sensor.

### Target coordinates

The pipeline is configured for lat=50.798056°N, lon=4.357500°E (Belgium). These are hardcoded in `cams_download3.py`, `interpolation1.py`, and `uvspec_template.inp`.
